## XML 文件的导入顺序与最后的关节顺序紧密相关
如果你导入顺序为 hand -> cube 那么，hand的关节会cube之前；如果顺序为 cube -> hand 那么 cube 的关节会在 hand 之前。
这是因为 MuJoCo 在加载 XML 时会按照顺序分配关节 ID，所以导入顺序会影响关节的最终排列顺序，请务必注意这一点。无论添加什么道具，都要确保手部模型始终是最开始导入的部分，以保证手部关节在关节数组中的正确位置。

## 异步阻塞错误
你可能会在训练过程中观察到一些异步阻塞错误。这通常是由于渲染线程和物理计算线程之间的同步问题引起的。不用担心，这是正常现象，不会影响训练过程。

## 关于 nconmax 和 njmax 的设置
nconmax (Max Number of Contacts):

含义：这是**接触点（Contacts）**的最大数量。
解释：比如指尖碰到方块，这是一个接触点；手掌碰到方块，这是另一个。如果你的场景里有一堆物体堆在一起，接触点会非常多。
作用：它决定了 data.contact 数组的长度。
njmax (Max Number of Constraints):

含义：这是**标量约束（Scalar Constraints）**的最大数量。
解释：约束 = 接触。一个接触点通常会产生多个约束。接触法向力（不让物体穿透）：1 个约束。摩擦力（切向力）：通常是 2 个约束（X轴和Y轴摩擦）。关节限制（Joint Limits）：如果关节转到了极限，也会产生约束。

经验法则：通常 njmax 的值应该是 nconmax 的 3到4倍（因为每个接触点大约对应 3-4 个约束方程）。
